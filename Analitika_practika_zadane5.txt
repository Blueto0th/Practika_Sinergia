Аналитический обзор WEB-приложения «Туризм» (Кейс-задача № 4)

В рамках производственной практики было разработано WEB-приложение для управления заказами туров на базе Python (Flask) и MySQL. Ниже представлен анализ приложения по девяти ключевым критериям качества.

1. Функциональность
Приложение реализует базовый CRUD-функционал: просмотр списка заказов, добавление нового заказа, удаление существующего. Данные корректно извлекаются из связанных таблиц (Отели, Услуги) и отображаются в удобном табличном виде. Все операции выполняются без ошибок. Функциональность полностью соответствует поставленной задаче.

2. Производительность
Приложение работает с небольшим объёмом данных (5–10 записей), поэтому скорость выполнения операций высокая. Запросы к БД выполняются за доли секунды. При увеличении количества заказов до тысяч записей потребуется оптимизация запросов (индексы, кэширование), но в текущем виде производительность достаточная.

3. Удобство использования (юзабилити)
Интерфейс интуитивно понятен: пользователь видит все заказы в виде таблицы, может добавить новый через форму вверху страницы и удалить ненужный нажатием на крестик. Сообщения об ошибках (например, при неверном вводе) выводятся стандартными средствами Flask. Для улучшения можно добавить подтверждение перед удалением (реализовано через confirm() в JavaScript).

4. Безопасность
На данном этапе приложение не требует аутентификации, так как предназначено для демонстрационных целей. В промышленной эксплуатации необходимо добавить авторизацию, защиту от SQL-инъекций (используется параметризованные запросы в MySQL Connector, что уже обеспечивает базовую защиту) и HTTPS.

5. Масштабируемость
Приложение построено на Flask — лёгком фреймворке, который позволяет легко расширять функционал (добавлять новые маршруты, формы, отчёты). Для масштабирования на большие нагрузки можно использовать WSGI-серверы (Gunicorn, uWSGI) и балансировщики.

6. Сопровождаемость
Код структурирован: маршруты вынесены в отдельные функции, подключение к БД — в отдельный метод. Комментарии присутствуют в ключевых местах. Для улучшения сопровождаемости можно добавить документацию и логирование.

7. Переносимость
Приложение написано на Python с использованием кроссплатформенных библиотек (Flask, MySQL Connector). Оно может быть запущено на Windows, Linux или macOS. Для переноса необходимо установить зависимости из requirements.txt и настроить подключение к БД.

8. Качество кода
Код соответствует стандартам PEP 8, функции названы осмысленно, используются параметризованные запросы для защиты от SQL-инъекций. Применён шаблонизатор Jinja2 для разделения логики и представления.

9. Тестирование
Приложение протестировано вручную: проверены все сценарии (просмотр, добавление, удаление). Ошибок не выявлено. Для дальнейшего развития рекомендуется добавить автоматические тесты с использованием pytest.

Выявленные проблемы и способы их решения:

Проблема	Способ решения
Отсутствие авторизации	Добавить систему регистрации/входа на основе Flask-Login
Нет валидации ввода	Добавить проверку типов данных и обязательных полей на серверной стороне
Ограниченный интерфейс	Добавить пагинацию при большом количестве заказов, фильтры по статусу
Отсутствие логирования	Внедрить модуль logging для отслеживания ошибок и действий пользователей
Нет редактирования заказов	Добавить форму редактирования и маршрут /edit/<id>
